## What is TailorHub Skills?

TailorHub Skills is a CLI tool that makes it easy to install, update, and manage AI skills for Claude Code and Codex. Skills are installed via `npx` without requiring a global installation.

## Installation

No installation required! Run commands directly with `npx`:

```bash
npx @tailorhub/skills@latest
```

<Note>
Requires Node.js 18 or higher
</Note>

## Quick Start

<Steps>
  <Step title="Install your first skill">
    Install a skill to both Claude Code and Codex (default behavior):

    ```bash
    npx @tailorhub/skills@latest add https://github.com/TailorHub-Mad/ai-skills/tailor-code-review
    ```

    The CLI will:
    - Download the skill from GitHub
    - Install it to `~/.claude/skills` and `~/.codex/skills`
    - Create the directories automatically if they don't exist
  </Step>

  <Step title="Restart your apps">
    After installation, restart Claude Code and/or Codex to apply changes.

    <Warning>
    Skills won't be available until you restart the corresponding app(s)
    </Warning>
  </Step>

  <Step title="Verify installation">
    Your skill is now ready to use! The CLI saves metadata in `.source.json` files to enable updates.
  </Step>
</Steps>

## Commands

### Add a skill

Install skills from GitHub repositories using the format:  
`https://github.com/<owner>/<repo>/<skill>`

<CodeGroup>
```bash Default (both targets)
npx @tailorhub/skills@latest add https://github.com/TailorHub-Mad/ai-skills/tailor-code-review
```

```bash Codex only
npx @tailorhub/skills@latest add https://github.com/TailorHub-Mad/ai-skills/tailor-code-review --target codex
```

```bash Claude Code only
npx @tailorhub/skills@latest add https://github.com/TailorHub-Mad/ai-skills/tailor-mermaid-to-drawio --target claude
```
</CodeGroup>

<Note>
Skills with subfolders (like `agents/` and `references/`) are downloaded recursively
</Note>

### Update skills

Keep your skills up to date by re-downloading them from their saved source.

<CodeGroup>
```bash Update all skills
npx @tailorhub/skills@latest update
```

```bash Update specific skill
npx @tailorhub/skills@latest update tailor-code-review
```

```bash Update for single target
npx @tailorhub/skills@latest update --target claude
```

```bash Update specific skill on Codex only
npx @tailorhub/skills@latest update tailor-mermaid-to-drawio --target codex
```
</CodeGroup>

<Tip>
The `update` command re-downloads all files in the skill, including nested folders, from the original GitHub source
</Tip>

### Remove skills

Delete installed skills from your system.

<CodeGroup>
```bash Remove from both targets
npx @tailorhub/skills@latest remove tailor-code-review
```

```bash Remove from Codex only
npx @tailorhub/skills@latest remove tailor-code-review --target codex
```

```bash Remove from Claude Code only
npx @tailorhub/skills@latest remove tailor-code-review --target claude
```
</CodeGroup>

<Note>
The `remove` command only deletes the specific skill folder, not the entire skills directory
</Note>

## Target Options

The CLI supports three target options:

| Target | Installs to | App |
|--------|-------------|-----|
| `claude` | `~/.claude/skills` | Claude Code |
| `codex` | `~/.codex/skills` | Codex |
| `both` | Both locations | Both apps |

<Tip>
If you don't pass `--target`, the default is `both`
</Tip>

## Partial Success Behavior

When using `--target both` (or omitting `--target`), one target may fail while the other succeeds.

- The command succeeds if **at least one** target succeeds
- The CLI prints a per-target result summary
- The command fails only if **all** requested targets fail

Example output:
```
Results:
  [claude] installed -> /home/user/.claude/skills/tailor-code-review/
  [codex] failed: HTTP 404 for https://api.github.com/...
Installation completed with partial success.
Restart Claude Code to apply changes.
```
